# META-HARNESS: Self-Bootstrapping Agent Infrastructure

## What This Is
A meta-harness that generates task-specific harnesses and agents from vague intent.
You give it a fuzzy goal. It figures out what to build, how to constrain it, and who should do it.

## The Compilation Pipeline
```
Vague Intent → [Interpreter] → Structured Task
Structured Task → [Harness Generator] → Task-Specific Harness
Task-Specific Harness → [Agent Factory] → Specialized Agents
Specialized Agents + Harness → [Orchestrator] → Execution
Execution Results → [Feedback] → Improved Meta-Harness
```

## Architecture
```
META.md                    ← You are here. The bootstrap DNA.
meta/
  interpreter.md           ← Intent → Structured Task Definition
  harness-generator.md     ← Task Definition → Harness (constraints, workflows, skills)
  agent-factory.md         ← Harness → Agent Configurations (roles, tools, scope)
  orchestrator.md          ← Agents + Harness → Execution Plan
templates/                 ← Reusable building blocks for harness generation
  web-app/                 ← Template: web application
  api-service/             ← Template: API/backend service
  data-pipeline/           ← Template: data processing pipeline
  content-system/          ← Template: content generation/management
  automation/              ← Template: workflow automation
generated/                 ← Output: generated harness+agent configurations
memory/                    ← Meta-level memory (compounds over time)
scripts/
  bootstrap.sh            ← Entry point: intent in, execution out
```

## How to Bootstrap
1. User provides vague intent (e.g., "I need a customer onboarding system")
2. Agent reads THIS file to understand the pipeline
3. Agent follows `meta/interpreter.md` to parse intent into structured task
4. Agent follows `meta/harness-generator.md` to generate a harness
5. Agent follows `meta/agent-factory.md` to create agent configurations
6. Agent follows `meta/orchestrator.md` to plan execution
7. Generated output goes to `generated/[project-name]/`
8. Execution begins with generated harness+agents
9. Mistakes feed back into `memory/` and improve future generations

## Meta-Rules (These Cannot Be Overridden)
1. No execution without interpretation — vague intent must be structured first
2. No agent without a harness — every agent operates within generated constraints
3. No constraint without a reason — every rule must trace to a task requirement
4. No completion without verification — generated harness must include verification
5. Every generation is logged — `memory/generation-log.md` tracks all outputs
6. Every failure improves the meta — mistakes in generation update templates
7. The meta-harness is itself a harness — it follows its own rules

## The Self-Application Paradox
This meta-harness was itself generated by an agent following harness principles.
It bootstraps itself: the rules that govern generated harnesses also govern the meta-harness.
If the meta-harness makes a mistake, the same feedback loop applies.
